# Decoded Ops Multi-Phase Implementation Plan

**Project:** Decoded Ops Website & Proposal System  
**Owner:** Craig Blackman  
**Created:** 2026-04-19  
**Status:** Ready to Execute (Phase 0 complete)

---

## Overview

Seven-phase strategic plan to build a scalable proposal system where website messaging flows into proposals, outreach, and templates. Phase 0 (brand consolidation) is complete. Phases 1-7 span knowledge base foundation → content optimization → proposal automation → outreach at scale → analytics-driven conversion → thought leadership → performance optimization.

**Total scope:** 32 tasks | 330 estimated hours | 7 phases

## Phase 3: Proposal Generation & Lead Capture
**Duration:** 3 weeks | **Effort:** 72 hours | **Depends on:** Phase 1

Automate proposal generation from website questionnaire. Capture leads into CRM. Send proposal previews via email.

### Tasks

**P3-001: Questionnaire Refinement** (High | 8h)
- Expand pricing questionnaire with business maturity assessment
- Add branching logic: segment A (large) vs. segment B (small business)
- Score responses: readiness for Diagnose/Advise/Deliver
- Link to proposal tiers
- Depends on: P1-001 (Knowledge Base)

**P3-002: Proposal Data Schema** (High | 6h)
- Design schema: map questionnaire answers → proposal sections → pricing tiers
- Define fields: company name, turnover, staff, current systems, pain points, timeline, budget
- Create mapping: which answers trigger which proposal tier?
- Document in `/knowledge-base/`
- Depends on: P3-001 (Questionnaire)

**P3-003: Dynamic Proposal Component** (High | 20h)
- Build React component: generates proposal preview from questionnaire answers
- Sections: executive summary, challenge statement, proposed approach, pricing tier, next steps, terms
- Include PDF export (use react-pdf or similar)
- Test all pricing paths (Diagnose/Advise/Deliver for both segments)
- Depends on: P3-002 (Schema)

**P3-004: Email Integration** (Medium | 12h)
- Create lead capture form (name, email, company)
- Send email confirmation with questionnaire summary
- Attach proposal preview as PDF
- Store answers in database for CRM sync
- Depends on: P3-003 (Proposal Component)

**P3-005: CRM Integration** (Medium | 16h)
- Integrate with Bitrix24 API
- Sync lead data to Bitrix24 contacts on form submission
- Store questionnaire answers in contact custom fields
- Set up pipeline trigger: move contact to "Website Lead" status
- Depends on: P3-004 (Email Integration)

### Verification Checklist
- [ ] Questionnaire branch logic working (segment A/B routing)
- [ ] Proposal component renders all 6 pricing paths correctly
- [ ] PDF export working and readable
- [ ] Lead form captures data and sends email
- [ ] Bitrix24 syncing contacts and questionnaire answers
- [ ] End-to-end test: submit questionnaire → receive proposal email → contact appears in Bitrix24

### Blockers & Dependencies
- Depends on: P1-001 (Knowledge Base for schema storage)
- Requires: Bitrix24 API key

## Phase 7: Performance & Accessibility
**Duration:** 2 weeks | **Effort:** 42 hours | **No dependencies**

Optimize website performance. Ensure accessibility (WCAG 2.1 AA). Mobile-first refinement.

### Tasks

**P7-001: Website Performance Audit** (Medium | 8h)
- Audit: page speed (Lighthouse), Core Web Vitals (LCP, FID, CLS), mobile performance
- Check: accessibility (WCAG 2.1 AA), SEO health (crawlability, indexability)
- Create performance report with prioritized fixes

**P7-002: Infrastructure Optimization** (Medium | 12h)
- Optimize images: next/image, WebP format, responsive sizes
- Implement caching: browser caching, CDN, server-side caching
- Enable compression: Gzip/Brotli, minify CSS/JS
- Target: <2s LCP, <100ms FID, <0.1 CLS on mobile

**P7-003: Accessibility Improvements** (Low | 10h)
- Fix WCAG violations: color contrast, keyboard navigation, screen reader compatibility
- Test with: axe DevTools, WAVE, keyboard-only navigation
- Ensure: form labels, alt text, heading hierarchy

**P7-004: Mobile-First Refinement** (Medium | 12h)
- Test on real devices (iPhone, Android)
- Improve: touch targets (48px minimum), mobile nav, form inputs, pricing page on small screens
- Test landscape orientation

### Verification Checklist
- [ ] Lighthouse score: 90+ on all pages
- [ ] Core Web Vitals: all green on mobile and desktop
- [ ] WCAG 2.1 AA: zero violations
- [ ] Accessibility audit: keyboard navigation, screen reader, color contrast all pass
- [ ] Mobile test: all features work on 375px and 768px viewports

### Blockers & Dependencies
- None — can run in parallel with earlier phases if needed
